Text Contrast

While designing with color in user interfaces, keep accessibility in mind. For optimal readability, text color should be $gray-dark against a white background. For minimum WCAG 2.0 compliance, use darken( $gray, 20% ) or darker for regular-weight text less than 18pt. When you need to create hierarchy in your design, take these guidelines into consideration (assuming text is on a white background):

  • For text that is non-essential but should still be readable, use $gray or darker (e.g. tags, sub-heads, timestamps, etc.).
  • Use caution when using colors lighter than $gray. These colors should only be used for elements that are not essential to be read by the user (e.g. disabled elements).
  • The WCAG 2.0 formula differentiates between text smaller than 18pt and text larger than 18pt (or text that is bold and larger than 14pt).

To check for WCAG compliance, click on the icon next to the color name.
