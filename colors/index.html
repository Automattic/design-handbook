Colors

They were the first thing to be documented internally in an attempt to achieve a more consistent Design Language for WordPress.com. Colors are one of the most notable aspects in brand recognition.

Text Contrast

While designing user interfaces, keep accessibility in mind. For optimal readability, text color should be $gray-dark against a white background. For minimum WCAG 2.0 compliance, use darken( $gray, 20% ) or darker for regular-weight text less than 18pt. When you need to create hierarchy in your design, take these guidelines into consideration (assuming text is on a white background):

  • For text that is non-essential but should still be readable, use $gray or darker (e.g. tags, sub-heads, timestamps, etc.).
  • Use caution when using colors lighter than $gray. These colors should only be used for elements that are not essential to be read by the user (e.g. disabled elements).
  • For bold text that's at least 14pt, the guidelines are less strict.

To check for WCAG compliance, click on the icon next to the color name.
